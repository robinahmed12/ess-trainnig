git init → Start tracking a folder with Git. git status → Check what files changed. git add file.txt → Prepare a file for saving (staging). git commit -m "message" → Save the changes with a message. git push → Upload changes to GitHub. git pull → Download the latest changes from GitHub. git clone URL → Copy someone’s project from GitHub. Create a new branch git branch branch-name Switch to a branch git checkout branch-name or git switch branch-name Create & switch in one step git checkout -b branch-name See all branches git branch Rename a branch git branch -m old-name new-name Merge a branch git merge branch-name Delete a branch (local) git branch -d branch-name Force delete a branch git branch -D branch-name Push branch to GitHub git push origin branch-name Pull branch from GitHub git fetch origin + git checkout branch-name Delete branch from GitHub git push origin --delete branch-name Switch to branch to merge INTO git checkout main Merge another branch into it git merge branch-name Cancel a merge in progress git merge --abort See merged branches git branch --merged See branches not merged git branch --no-merged Push merged changes to GitHub git push origin main Fast-forward Merge Main has no new commits, moves forward A-B → A-B-C-D Three-way Merge Both branches have new commits, creates merge commit A-B-E + C-D → M 1. Update main git checkout main git pull origin main 2. Merge feature git merge feature-branch 3. Push changes git push origin main (1) git clone ← Get project from GitHub | (2) git pull ← Always update to latest version | (3) git checkout -b feature-login ← Create a new branch for your work | (4) Work on code (edit files) | (5) git status ← Check changes (6) git add . ← Stage changes (7) git commit -m "Added login feature" ← Save changes locally | (8) git push origin feature-login ← Upload branch to GitHub | (9) Create Pull Request (on GitHub) | (10) Team reviews & merges to main
